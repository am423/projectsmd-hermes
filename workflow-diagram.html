ProjectsMD Workflow

Single-file project management for agent-human collaboration

The Session Loop

  • • Start: projectsmd status → next → read state
  • • Work: task done • decide • discover
  • • End: projectsmd session (wrap-up wizard)
  • • Current State updated every session
  • • Next session picks up exactly where you left off

Phase Lifecycle

  • • DEFINE → DESIGN → BUILD → VERIFY → SHIP
  • • projectsmd phase transition verify
  • • Evolution checklist at each transition
  • • Requirements promoted/retired automatically
  • • "What This Is" checked for drift

Requirements Lifecycle

  • • Active = hypotheses (current scope)
  • • Validated = proven (shipped + valuable)
  • • Out of Scope = rejected (with reasoning)
  • • Not just done/not done — three tiers
  • • Prevents silent scope creep
